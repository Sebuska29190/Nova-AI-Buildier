**Cel:** Zamknąć największe luki względem Hermes Agent

**Szacowany czas:** 5-7 dni agenta

## Faza 1: Browser Automation (🔴 HIGH — 1.5 dnia)

Port z Hermesa `browser_camofox.py` — stealth browser z fingerprinting + screenshot, click, type, extract.

### Pliki do utworzenia

| Plik | Opis |
|------|------|
| `packages/core/src/browser/engine.ts` | Playwright/Puppeteer wrapper — stealth config, fingerprinting, CDP |
| `packages/core/src/browser/fingerprint.ts` | User-agent rotacja, viewport, WebGL, canvas, audio fingerprint spoofing |
| `packages/core/src/browser/actions.ts` | Click, type, screenshot, extract, scroll, wait, evaluate |

### API design
```typescript
browser_navigate(url: string, options?: { width, height, userAgent })
browser_click(selector: string, options?: { timeout })
browser_type(selector: string, text: string)
browser_screenshot(fullPage?: boolean) => string (path)
browser_extract(selector: string) => string (text/HTML)
browser_evaluate(script: string) => any
browser_list_tabs() => TabInfo[]
browser_close()
```

### Narzędzia (8)
`browser_navigate`, `browser_click`, `browser_type`, `browser_screenshot`, `browser_extract`, `browser_evaluate`, `browser_list_tabs`, `browser_close`

### Zależności
- `npm install playwright` (lub `puppeteer`)
- Opcjonalnie: `playwright install chromium` (first-run)

### Hermes reference
- `browser_camofox.py` ~400 linii
- `computer_use_tool.py` ~200 linii (częściowo pokrywa się)

## Faza 3: Email Bridge (🟡 MEDIUM — 1.5 dnia)

IMAP + SMTP jako pełny channel w Novie. Odbieranie, wysyłanie, forwarding, wątkowanie.

### Pliki do utworzenia

| Plik | Opis |
|------|------|
| `packages/core/src/channel/email.ts` | Email bridge — IMAP polling + SMTP send |
| `packages/core/src/channel/templates/email-reply.md` | Szablon odpowiedzi email |
| `packages/core/src/email/` | Katalog dla helperów |

### API design
```typescript
interface EmailConfig {
  imap: { host, port, user, password, tls }
  smtp: { host, port, user, password, tls }
  pollingInterval?: number  // ms, default 30000
}

email_send(to: string, subject: string, body: string, options?: { cc, bcc, attachments })
email_list(folder?: string, limit?: number) => Email[]
email_read(uid: number) => Email (full)
email_reply(uid: number, body: string)
email_search(query: string) => Email[]
email_list_folders() => string[]
```

### Narzędzia (6)
`email_send`, `email_list`, `email_read`, `email_reply`, `email_search`, `email_list_folders`

### Zależności
- IMAP: `node-imap` lub `mailparser` (npm)
- SMTP: `nodemailer` (npm)

### Hermes reference
- `email_tool.py` ~300 linii
- `send_message_tool.py` ~100 linii

## Faza 4: WhatsApp Bridge (🟡 MEDIUM — 1 dzień)

WhatsApp Web przez Puppeteer/Playwright (jak whatsapp-web.js) lub Business API.

### Opcje implementacji

| Opcja | Zalety | Wady |
|-------|--------|------|
| **whatsapp-web.js** | Dojrzały, events, media | Wymaga Chrome, wrażliwy na zmiany WA Web |
| **Baileys** (node) | Lekki, WebSocket, bez Chrome | WebSocket protocol reverse-engineered |
| **Business API** | Oficjalny, stabilny | Wymaga Meta Business verification |
| **Playwright** (własny) | Pełna kontrola | Więcej kodu, kruchy |

### Rekomendacja: **whatsapp-web.js** (najszybsza integracja)

```typescript
whatsapp_connect() => QR code (do zeskanowania)
whatsapp_send(phone: string, message: string)
whatsapp_send_image(phone: string, caption: string, imagePath: string)
whatsapp_list_chats(limit?: number) => Chat[]
whatsamp_get_messages(chatId: string, limit?: number) => Message[]
whatsapp_disconnect()
```

### Narzędzia (5)
`whatsapp_connect`, `whatsapp_send`, `whatsapp_send_image`, `whatsapp_list_chats`, `whatsapp_get_messages`

### Zależności
- `npm install whatsapp-web.js` (pulluje Chrome)

## Faza 5: Signal Bridge (🟡 MEDIUM — 0.5 dnia)

Signal Messenger przez `signal-cli` (REST API lokalnego daemona) lub official Signal API.

### Opcje implementacji

| Opcja | Zalety | Wady |
|-------|--------|------|
| **signal-cli** | Działa, REST API przez dbus/JSON | Wymaga Javy, pierwsze połączenie telefoniczne |
| **signal-bot** (node) | Czysty Node, prosty | Mniej dojrzały |

### Rekomendacja: **signal-cli** (bo już sprawdzony w Hermes)

```typescript
signal_connect(phoneNumber: string)
signal_send(recipient: string, message: string)
signal_receive(limit?: number) => Message[]
signal_list_groups() => Group[]
```

### Narzędzia (3)
`signal_connect`, `signal_send`, `signal_receive`

### Zależności
- Java Runtime (JRE 17+)
- `signal-cli` binary pobrany i skonfigurowany
- Pierwsze połączenie: wysyła SMS/call z kodem weryfikacyjnym
